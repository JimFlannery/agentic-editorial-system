# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Instead, open a GitHub Security Advisory on this repository:
**Security → Advisories → Report a vulnerability**

Include as much detail as possible: steps to reproduce, affected versions, and potential impact. You will receive a response within 5 business days. If the issue is confirmed, a fix will be prioritised and you will be credited in the release notes unless you prefer to remain anonymous.

## Supported Versions

Only the latest release on the `main` branch receives security fixes at this stage of the project.

## Scope

In scope: the AgenticES application code, its API routes, authentication flows, and data access patterns.

Out of scope: vulnerabilities in upstream dependencies (please report those to the dependency maintainer directly), and findings from automated scanners without proof of exploitability.
