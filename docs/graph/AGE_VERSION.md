## Parameter-passing approaches

| Approach | Syntax | Result |
|----------|--------|--------|
| A — no params, inline literal | `$$ MATCH (n {id: 'node-1'}) $$` | PASS (1 row) |
| B — `$1::agtype` blob, ref as `$paramName` | `$$ MATCH (n {id: $id}) $$, $1::agtype` | PASS (1 row) |
| C — `$params.id` dot-notation | `$$ MATCH (n {id: $params.id}) $$, $1::agtype` | PASS — no error, but 0 rows |

Approach C does not throw but matches nothing — `$params.id` is silently treated as
an unbound reference in AGE 1.7.0 rather than dot-navigating the parameter blob.
Do not use this syntax.

## Decision

**Use Approach B.** `AGESession._wrap()` in `lib/graph/repository.ts` uses this form.

Parameters are passed as a single `agtype` JSON blob in SQL `$1`. Inside Cypher,
reference individual keys directly as `$paramName` — **not** `$params.paramName`.

Example:
```sql
SELECT * FROM cypher('t_tenant', $$ MATCH (n {id: $id}) RETURN n $$, $1::agtype) AS (n agtype)
-- $1 = '{"id": "abc123"}'
```

## agtype parser

`parseAgtypeValue()` in `lib/graph/pool.ts` correctly flattens vertex properties.

Real output from AGE 1.7.0:
```
{"id": 844424930131969, "label": "TestNode", "properties": {"id": "node-1", "label": "hello"}}::vertex
```

Parsed result:
```json
{ "id": "node-1", "label": "hello", "_id": 844424930131969, "_label": "TestNode" }
```

Properties from the `properties` map are promoted to the top level. The AGE internal
numeric id is preserved as `_id`; the label as `_label`.
